Planning Document

Scope: To develop a Java Banking Applet that enables access of both customers and bank representatives.

The customers can

  • perform transactions such as deposit and withdrawal

  • create a login account (sign up and sign in)

  • login as a customer and make transactions

  •  update their profile

  • view their transaction history

The representatives can

  • search customers

  • obtain customer details (except those like password, SSN which are the customer's privacy)

  • view transactions made by customers

 

Primary Actor: User who loads the page to view and use the Banking Applet.

Stakeholders and Interests:

-Person who loads the web page

      Is interested to successfully load and use the Banking Applet

Pre-conditions:

Internet connection is established.

The web page is loaded successfully.

The applet is loaded successfully.

Success Guarantee - Post Conditions:

The user is able to successfully load the banking applet.

The customers are able to

  • perform transactions such as deposit and withdrawal

  • create a login account (sign up and sign in)

  • login as a customer and make transactions

  •  update their profile

  • view their transaction history

The representatives are able to

  • search customers

  • obtain customer details (except those like password, SSN which are the customer's privacy)

  • view transactions made by customers

 

Extensions or Alternative Flows:

1. At any time, the internet connection is lost.

    a. Error message displayed requesting the user to check his internet connection.

2. The user clicks the back button before the program is loaded fully.

    b. The program terminates and the previous web page is displayed.

 

Design

To satisfy the stated requirements, it requires 6 pages.

  1. Login Page

  2. Registration Page

  3. Transaction Page

  4. Transaction History

  5. Profile Update / View

  6. Search Page

Planning Docoment

Scope: Finding the factorial of a number; calculating the average of a list of integers, which are stored in a text file and whose file name is given as a command line argument; To generate email ids of students using their first name, last name and social security number which are stored in a file; To create a packageand store all the above files inside the package,

Level : User goal

Primary Actor: User who loads the page to find the factorial, average or to generate email ids.

Stakeholders and Interests:

-Person who loads the web page

      Is interested to successfully load the applets and find the factorial, avaerage or generate email ids.

Pre-conditions:

Internet connection is established.

The web page is loaded successfully.

A file with first name, last name and email ids is stored and the right path is feeded.

A file with inputs for the average program is stored in a location.

Success Guarantee - Post Conditions:

The factorial program, average program and generating email ids program are all run successfully and the reults are displayed to the user.

Main Success Scenario (or Basic Flow):

1. User clicks on the link directing to an operation.

2. The corresponding web page opens.

3. The contents of the applet are displayed and the user is able to feed the inputs and the desired outputs are generated.

Extensions or Alternative Flows:

1. At any time, the internet connection is lost.

a. Error message displayed requesting the user to check his internet connection.

2. The user clicks the back button before the program is loaded fully.

    b. The program terminates and the previous web page is displayed.

3. The input file path is wrong or the file does not exist for the average and email generator programs.

    c. Error message is displayed to the user.

4. User feeds in a non-integer value for the average calculation program.

    d. Error message is displayed to the user.
